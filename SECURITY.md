# Security Policy

Do not report exploitable vulnerabilities in public issues.

When reporting a security concern, include:

- the affected file or workflow;
- the realistic impact;
- steps to reproduce or demonstrate the issue;
- a proposed mitigation, when known.

Never include credentials, API keys, access tokens, private project links, customer data, or other secrets in examples, issues, or pull requests.

This repository contains educational material and reusable workflows. It does not replace a professional security review of a production application.
